Dependency Injection Pattern

Dependency Injection

Description

Dependency Injection is a technique where one object supplies the dependencies of another object, rather than the dependent object creating them itself. It promotes loose coupling and testability.

Advantages

  • Eases unit testing by injecting mock dependencies.
  • Reduces coupling between objects, making the code more modular.
  • Allows easy swapping of dependencies at runtime or via configuration.

Use in Node.js

Used in frameworks like NestJS to manage services or modules, or in Express apps for injecting repositories into controllers.

Notes

DI can be implemented via constructor, setter, or interface injection. In Node.js, libraries like InversifyJS provide IoC containers for advanced DI.

Tips

  • Use interfaces for dependencies to ensure abstraction.
  • Avoid service locator anti-pattern; prefer explicit injection.
  • In TypeScript, use generics for type-safe DI containers.

Additional Information

DI is a core principle in SOLID, and in Node.js, it's essential for scalable applications, often combined with IoC containers like Awilix.

Code Example


interface Logger { log(message: string): string; }
class ConsoleLogger implements Logger { log(message: string): string { return `Console: ${message}`; } }

class UserService {
  constructor(private logger: Logger) {}
  createUser(name: string): string { return this.logger.log(`User ${name} created`); }
}
          

function UserService(logger) {
  this.logger = logger;
}
UserService.prototype.createUser = function(name) {
  return this.logger.log("User " + name + " created");
};
