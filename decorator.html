Decorator Pattern

Decorator

Description

The Decorator pattern allows adding new functionality to an existing object dynamically by wrapping it with one or more decorator objects. It's useful for extending behavior without subclassing.

Advantages

  • Allows flexible extension of object functionality at runtime.
  • Maintains the Open/Closed Principle by allowing extension without modification.
  • Reduces the need for multiple subclasses, avoiding explosion of classes.

Use in Node.js

Common in NestJS for adding logging or authentication to functions, or in Express for decorating middleware functions.

Notes

Decorators in TypeScript are experimental but powerful for metadata. In Node.js, they're used in AOP libraries like Aspect.js.

Tips

  • Ensure decorators preserve the original method's return type.
  • Use composition over inheritance for multiple decorators.
  • Enable experimentalDecorators in tsconfig.json for TS support.

Additional Information

The Decorator pattern is the foundation for TypeScript decorators, used in frameworks like Angular for component decoration.

Code Example


function Logger(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
  const originalMethod = descriptor.value;
  descriptor.value = function (...args: any[]) {
    return `Calling ${propertyKey} with ${args.join(", ")}. Result: ${originalMethod.apply(this, args)}`;
  };
}

class Calculator {
  @Logger
  add(a: number, b: number): number { return a + b; }
}
          

function Logger(originalMethod) {
  return function (...args) {
    return `Calling with ${args.join(", ")}. Result: ${originalMethod.apply(this, args)}`;
  };
}

class Calculator {
  add(a, b) { return a + b; }
}
Calculator.prototype.add = Logger(Calculator.prototype.add);
