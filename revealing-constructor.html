Revealing Constructor Pattern

Revealing Constructor

Description

The Revealing Constructor pattern is a variation of the constructor pattern that reveals the object's API during construction, allowing for configuration before the object is fully instantiated. It's useful for creating immutable objects with customizable interfaces.

Advantages

  • Allows customization during creation without modifying the object later.
  • Ideal for immutable objects, ensuring thread-safety in concurrent environments.
  • Provides fine-grained control over the object's public API.

Use in Node.js

In Node.js, it's used for creating immutable buffers or server configurations, such as building read-only configuration objects for microservices.

Notes

This pattern is less common in modern JavaScript but shines in scenarios requiring immutability, like in Redux state management or configuration loaders.

Tips

  • Use Object.freeze() to enforce immutability after construction.
  • Pass a callback to the constructor for custom initialization.
  • Avoid overusing it; prefer Builder for very complex constructions.

Additional Information

Popularized in JavaScript modules, this pattern is seen in libraries like Lodash for creating utility functions with custom configurations.

Code Example


class ImmutableBuffer {
  private buffer: Buffer;
  constructor(size: number, initializer: (buffer: Buffer) => void) {
    this.buffer = Buffer.alloc(size);
    initializer(this.buffer);
    Object.freeze(this.buffer);
  }
  read(): Buffer { return this.buffer; }
}
          

function ImmutableBuffer(size, initializer) {
  const buffer = Buffer.alloc(size);
  initializer(buffer);
  Object.freeze(buffer);
  return {
    read: () => buffer
  };
}
