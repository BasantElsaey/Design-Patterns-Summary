Proxy Pattern

Proxy

Description

The Proxy pattern provides a surrogate or placeholder for another object to control access to it. It's used to add functionality like lazy loading, validation, or caching without changing the original object.

Advantages

  • Adds functionality without modifying the original object (Open/Closed Principle).
  • Supports caching, access control, and lazy initialization.
  • Allows separation of additional logic from the core object.

Use in Node.js

Used for resource access control or caching in applications, such as proxying database queries to add logging or rate limiting.

Notes

ES6 introduced the built-in Proxy object, making implementation easier. In Node.js, it's useful for virtualizing file systems or API calls.

Tips

  • Use the built-in Proxy for dynamic proxies in modern JS.
  • Ensure the proxy delegates all methods to the real object to avoid breaking functionality.
  • Combine with Reflect API for transparent proxying.

Additional Information

The Proxy pattern is powerful for AOP (Aspect-Oriented Programming) in Node.js, often used in libraries like Express middleware proxies.

Code Example


interface Subject { request(): string; }

class RealSubject implements Subject {
  request(): string { return "RealSubject: Handling request"; }
}

class ProxySubject implements Subject {
  private realSubject: RealSubject;
  constructor() { this.realSubject = new RealSubject(); }
  request(): string {
    return `Proxy: Logging before request\n${this.realSubject.request()}`;
  }
}
