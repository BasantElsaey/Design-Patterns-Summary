Singleton Pattern

Singleton

Description

The Singleton pattern ensures that a class has only one instance and provides a global point of access to it. It's useful for managing shared resources like configuration or database connections.

Advantages

  • Reduces resource consumption by reusing a single instance.
  • Provides a unified access point for shared resources.
  • Ideal for managing global state or shared resources across the application.

Use in Node.js

Common in managing database connections like MongoDB or global app configurations, ensuring only one connection pool is used.

Notes

In Node.js, Singletons are naturally supported due to the single-threaded nature, but be cautious in clustered environments where multiple processes may create multiple instances.

Tips

  • Use lazy initialization to create the instance only when needed.
  • Consider dependency injection as an alternative to avoid global state issues.
  • Test by mocking the singleton instance for unit tests.

Additional Information

Singletons can be controversial due to global state, but in Node.js, they're useful for logging services or cache managers like Redis clients.

Code Example


class Database {
  private static instance: Database;
  private constructor() { console.log("Database connection initialized"); }
  public static getInstance(): Database {
    if (!Database.instance) Database.instance = new Database();
    return Database.instance;
  }
  query(sql: string): string { return `Executing query: ${sql}`; }
}
          

const Database = (function() {
  let instance;
  function createInstance() {
    const object = new Object("I am a singleton...");
    return object;
  }
  return {
    getInstance: function() {
      if (!instance) {
        instance = createInstance();
      }
      return instance;
    }
  };
})();
