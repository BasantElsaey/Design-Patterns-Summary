Adapter Pattern

Adapter

Description

The Adapter pattern allows objects with incompatible interfaces to collaborate by wrapping an old interface with a new one. It's used to make legacy code work with modern systems.

Advantages

  • Enables integration of libraries or systems with incompatible interfaces.
  • Reduces the need to rewrite legacy code, saving time and effort.
  • Promotes code reuse by adapting existing components.

Use in Node.js

Used to integrate legacy libraries with modern systems, like using LevelUP with a filesystem API or adapting third-party APIs.

Notes

Adapters are two-way; consider bidirectional adapters for full compatibility. In Node.js, they're common in migration tools.

Tips

  • Implement adapters as classes that implement the target interface.
  • Use composition to delegate calls to the adaptee.
  • Test adapters thoroughly to ensure all methods are mapped correctly.

Additional Information

The Adapter pattern is essential for microservices in Node.js, often used in gRPC adapters for REST APIs.

Code Example


interface NewUser { fullName: string; }
interface OldUser { firstName: string; lastName: string; }

class OldUserService {
  getUser(): OldUser { return { firstName: "Ali", lastName: "Ahmed" }; }
}

class UserAdapter implements NewUser {
  private oldUser: OldUser;
  constructor(oldUser: OldUser) { this.oldUser = oldUser; }
  get fullName(): string { return `${this.oldUser.firstName} ${this.oldUser.lastName}`; }
}
          

function OldUserService() {
  this.getUser = () => ({ firstName: "Ali", lastName: "Ahmed" });
}

function UserAdapter(oldUser) {
  this.get fullName() { return oldUser.firstName + " " + oldUser.lastName; }
}
