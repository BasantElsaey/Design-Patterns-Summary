Template Pattern

Template

Description

The Template pattern defines the skeleton of an algorithm in a method, deferring some steps to subclasses. It allows subclasses to redefine certain steps of an algorithm without changing the algorithm's structure.

Advantages

  • Ensures consistency in algorithm structure while allowing customization.
  • Allows customization of steps without altering the overall structure.
  • Facilitates code reuse by defining common steps in the base class.

Use in Node.js

Used for configuration management or customizable data processing, such as template methods for file processing pipelines.

Notes

Template is behavioral but creational in nature. In Node.js, it's used in build tools like Gulp for task pipelines.

Tips

  • Mark customizable methods as abstract in the base class.
  • Use protected access for template methods to allow subclass access.
  • Avoid deep inheritance hierarchies; prefer composition where possible.

Additional Information

Template is a classic pattern in framework design, seen in Node.js testing libraries like Mocha for test runners.

Code Example


abstract class DataProcessor {
  process(): string {
    return `${this.step1()}\n${this.step2()}`;
  }
  protected abstract step1(): string;
  protected abstract step2(): string;
}

class CSVProcessor extends DataProcessor {
  protected step1(): string { return "Reading CSV"; }
  protected step2(): string { return "Processing CSV data"; }
}
          

function DataProcessor() {}
DataProcessor.prototype.process = function() {
  return this.step1() + "\n" + this.step2();
};

function CSVProcessor() {}
CSVProcessor.prototype = Object.create(DataProcessor.prototype);
CSVProcessor.prototype.step1 = function() { return "Reading CSV"; };
CSVProcessor.prototype.step2 = function() { return "Processing CSV data"; };
