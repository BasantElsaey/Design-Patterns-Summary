Command Pattern

Command

Description

The Command pattern turns a request into a stand-alone object containing all information about the request. This allows for parameterization of clients with different requests and supports undoable operations.

Advantages

  • Allows undoing or redoing operations by storing commands in a history.
  • Reduces coupling between sender and receiver of a request.
  • Supports queuing or logging commands for later execution.

Use in Node.js

Used in task or command management systems, like remote control systems or undo/redo in text editors built with Node.js.

Notes

Command is the basis for the Task pattern in asynchronous programming. In Node.js, it's used in CLI tools for command queuing.

Tips

  • Implement undo by storing reverse operations or state snapshots.
  • Use macros to compose multiple commands into one.
  • Combine with Observer for notifying about command execution.

Additional Information

Command is foundational for event sourcing in Node.js, seen in libraries like EventStore for audit trails.

Code Example


interface Command { execute(): string; }

class Light {
  turnOn(): string { return "Light is ON"; }
}

class TurnOnCommand implements Command {
  private light: Light;
  constructor(light: Light) { this.light = light; }
  execute(): string { return this.light.turnOn(); }
}

class RemoteControl {
  submit(command: Command): string { return command.execute(); }
}
          

function Light() {
  this.turnOn = () => "Light is ON";
}

function TurnOnCommand(light) {
  this.execute = () => light.turnOn();
}

function RemoteControl() {
  this.submit = (command) => command.execute();
}
