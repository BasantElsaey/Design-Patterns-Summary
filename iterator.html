Iterator Pattern

Iterator

Description

The Iterator pattern provides a way to access elements of a collection sequentially without exposing its underlying representation. It supports protocols like iterator and iterable for consistent traversal.

Advantages

  • Simplifies navigation through complex data structures.
  • Supports safe and consistent iteration without exposing internal details.
  • Compatible with JavaScript features like for...of loops.

Use in Node.js

Used for iterating over user lists or processing data streams, such as in Readable streams or custom collections.

Notes

ES6 iterators are native; use Symbol.iterator for custom iterables. In Node.js, they're key for async iterators in streams.

Tips

  • Implement the iterator protocol with next() method returning { value, done }.
  • Use generators for lazy iteration to save memory.
  • Handle errors in iterators to prevent breaking the loop.

Additional Information

Iterators are foundational for async generators in Node.js, used in libraries like RxJS for observable sequences.

Code Example


interface Iterator { next(): T | null; hasNext(): boolean; }

class UserCollection {
  private users: string[];
  constructor(users: string[]) { this.users = users; }
  createIterator(): Iterator {
    let index = 0;
    return {
      next: () => (index < this.users.length ? this.users[index++] : null),
      hasNext: () => index < this.users.length
    };
  }
}
          

function UserCollection(users) {
  this.users = users;
}
UserCollection.prototype.createIterator = function() {
  let index = 0;
  return {
    next: () => (index < this.users.length ? this.users[index++] : null),
    hasNext: () => index < this.users.length
  };
};
